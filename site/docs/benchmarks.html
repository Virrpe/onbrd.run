Running Benchmarks

Execute benchmarks from the project root using our dedicated runner script:

# Run all benchmarks node scripts/run-benchmarks.mjs --seed 12345 --out benchmarks/results.json # Validate results against ground truth node scripts/validate-benchmarks.mjs --in benchmarks/results.json --out benchmarks/eval.json # Generate summary report node scripts/report-benchmarks.mjs # Run specific category node scripts/run-benchmarks.mjs --category saas --seed 12345 # Generate new fixture node scripts/gen-fixture.mjs saas onboarding-wizard-013

Scoring System

Benchmarks use deterministic scoring with the following heuristic weights:

  • H-CTA-ABOVE-FOLD (25%): Call-to-action visibility above the fold
  • H-STEPS-COUNT (20%): Number of steps in the onboarding process
  • H-COPY-CLARITY (20%): Readability and clarity of copy
  • H-TRUST-MARKERS (20%): Presence of trust-building elements
  • H-PERCEIVED-SIGNUP-SPEED (15%): Estimated time to complete signup

Validation Metrics

Benchmarks v0.2 includes validation against ground truth data with the following metrics:

  • Macro-F1 Score: ≥ 0.75 target for boolean checks
  • Out-of-band Scenarios: ≤ 10% per category
  • Global Calibration R²: ≥ 0.70 (predicted vs ground-truth)
  • Determinism: Variance 0.000000 across 3 seeds

Deterministic Scoring

All benchmarks use a seeded random number generator and fixed timestamps to ensure reproducible results:

  • Fixed Seed: 12345 (configurable via --seed parameter)
  • Fixed Timestamp: 2021-01-01T00:00:00.000Z
  • Deterministic PRNG: Linear congruential generator
  • Reproducible Results: Same seed produces identical scores
  • Validation: Ground truth comparison with macro-F1 metrics

Reproducibility Commands

Execute the following commands to reproduce these validation results:

# 1) TRAIN / none node scripts/run-benchmarks.mjs --seed 12345 --set train --perturb-dom none mv -f benchmarks/results.json benchmarks/results.train.none.json node scripts/validate-benchmarks.mjs --in benchmarks/results.train.none.json --out benchmarks/eval.train.none.json node scripts/report-benchmarks.mjs --results benchmarks/results.train.none.json --eval benchmarks/eval.train.none.json --out benchmarks/SUMMARY.train.none.md --context "train | perturb=none" # 2) TRAIN / light node scripts/run-benchmarks.mjs --seed 12345 --set train --perturb-dom light mv -f benchmarks/results.json benchmarks/results.train.light.json node scripts/validate-benchmarks.mjs --in benchmarks/results.train.light.json --out benchmarks/eval.train.light.json node scripts/report-benchmarks.mjs --results benchmarks/results.train.light.json --eval benchmarks/eval.train.light.json --out benchmarks/SUMMARY.train.light.md --context "train | perturb=light" # 3) HOLDOUT / none node scripts/run-benchmarks.mjs --seed 12345 --set holdout --perturb-dom none mv -f benchmarks/results.json benchmarks/results.holdout.none.json node scripts/validate-benchmarks.mjs --in benchmarks/results.holdout.none.json --out benchmarks/eval.holdout.none.json node scripts/report-benchmarks.mjs --results benchmarks/results.holdout.none.json --eval benchmarks/eval.holdout.none.json --out benchmarks/SUMMARY.holdout.none.md --context "holdout | perturb=none" # 4) FALSIFICATION (shuffle labels on TRAIN/NONE results) node scripts/validate-benchmarks.mjs --in benchmarks/results.train.none.json --shuffle-labels=true --out benchmarks/eval.train.shuffle.json node scripts/report-benchmarks.mjs --results benchmarks/results.train.none.json --eval benchmarks/eval.train.shuffle.json --out benchmarks/SUMMARY.train.shuffle.md --context "train | falsification"
